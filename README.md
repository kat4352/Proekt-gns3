# Отказоустойчивая корпоративная сеть на оборудовании Cisco (VLAN & HSRP)

Реализация архитектуры сети с разделением пользователей по изолированным виртуальным сетям (VLAN) и обеспечением высокой доступности шлюзов по умолчанию.

## 🛠 Технологический стек
* **Эмуляция:** GNS3
* **Оборудование:** Cisco vIOS-Router, Cisco vIOS-L2
* **Протоколы и технологии:** VLAN (IEEE 802.1Q), Router-on-a-Stick, HSRP, OSPF, NAT

## 📊 План адресации (Addressing Plan)

| Сеть (VLAN) | Имя | Виртуальный шлюз (HSRP) | Адрес DSW1 | Адрес DSW2 |
| :--- | :--- | :--- | :--- | :--- |
| **VLAN 10** | ADMINS | 192.168.10.1 /24 | 192.168.10.2 | 192.168.10.3 |
| **VLAN 20** | USERS | 192.168.20.1 /24 | 192.168.20.2 | 192.168.20.3 |

## ⚡ Журнал траблшутинга (Извлеченные уроки)

В процессе развертывания лабораторной работы были успешно решены следующие технические проблемы:

### 1. Ошибка синтаксиса `vlan 10` на уровне распределения
* **Проблема:** При попытке создания баз данных VLAN устройство выдавало синтаксическую ошибку `^` на команду `vlan`.
* **Причина:** Анализ вывода `show version` показал, что на схеме был запущен чистый образ маршрутизатора (`vIOS-Router`), не поддерживающий L2-функционал.
* **Решение:** Вместо классической Inter-VLAN маршрутизации через SVI была успешно реализована архитектура **Router-on-a-Stick** с разделением физического интерфейса `gi0/0` на логические субпорты (`gi0/0.10` и `gi0/0.20`) и инкапсуляцией трафика по международному стандарту **IEEE 802.1Q (dot1q)**.

### 2. Блокировка консоли при опечатках в CLI
* **Проблема:** Случайный ввод неверных команд (например, `jq`, `qq`) приводил к зависанию терминала на 1-2 минуты с сообщением `Translating...`.
* **Причина:** Включенный по умолчанию механизм поиска неизвестных команд через DNS-запросы.
* **Решение:** Процесс принудительно прерван комбинацией клавиш `Ctrl+Shift+6`. Для предотвращения повторения проблемы на всех узлах отключен поиск через команду `no ip domain-lookup`.
